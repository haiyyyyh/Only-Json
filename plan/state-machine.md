# 思路梳理

## json中的几种结构

### 特殊字面量
- `true`

### 数字

#### 检查:

一个合法的数字字面量可以是这样的
```
-8902.067e+23
```

检查状态机的状态表

|state|  0  | 1~9 | +/- | e/E |  .  |
|-----|-----|-----|-----|-----|-----|
|first|  0  | int | sig | err | err |
|  0  | err | err | err |  E  | dec |
| int |skip |skip | err |  E  | dec |
| dec |skip |skip | err |  E  | err |
|  E  |skip |skip |Esig | err | err |

伪代码解释

- 开始时
    - 遇到0
        - 后面是'.'或'e'则继续解析
        - 否则报错
    - '+'或'-'
        - 设置符号位
    - 1~9: 跳出前置特殊判断, 正常跑
- 在整形状态
    - 0~9跳过
    - '.'进入小数状态
    - 'e'进入科学记数法
    - 其他, 结束
- 在小数状态
    - 0~9跳过
    - 'e'进入科学记数法
    - '.'报错
    - 其他, 结束
        - 结束时判断结尾是不是'.'报错
        - 解析数字
- 在科学记数法状态
    - 0~9跳过
    - +/-跳过(设置标志位)
    - '.'报错
    - 'e'报错
    - 其他, 结束
        - 结束时判断结尾是不是'e'或者'.'报错
        - 解析数字

#### 计算:

接受几个状态: 正负性, 是否浮点, 是否科学记数法, 指数为正负性

### 字符串
- 遇到`"`, 开始识别字符串, 直到行尾, 忽略转义的\", 如果未匹配""则报错
- 字符串解码, 转义的\uXXXX为UTF-16, 以及一些特殊的转义符, 对非法转义符报错

### 对象
- 遇到`{`开始解析对象, `"str" : value`格式, 可有多组, 后跟随`,`, 最终都以`}`结尾, 期间忽略空白

    - 期间出现未预期的字符, 格式错误, 报错
    - 扫到文件尾仍未闭合的`{}`, 报错
    - 键或值出现的解析错误, 复用并返回 

### 数组
- 遇到`[`开始解析数组, 多个value, 中间`,`分隔, 最终以`]`结尾, 期间忽略空白
    - 未预期字符, 格式错误, 报错
    - 扫到文件尾仍未闭合的`[]`, 报错
    - 值出现解析错误, 复用并返回
