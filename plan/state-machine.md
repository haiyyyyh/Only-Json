### 数字

#### 检查:

一个合法的数字字面量可以是这样的
```
-8902.067e+23
```

检查状态机的状态表

|state|  0  | 1~9 | +/- | e/E |  .  |
|-----|-----|-----|-----|-----|-----|
|first|  0  | int | sig | err | err |
|  0  | err | err | err |  E  | dec |
| int |skip |skip | err |  E  | dec |
| dec |skip |skip | err |  E  | err |
|  E  |skip |skip |Esig | err | err |

伪代码解释

- 开始时
    - 遇到0
        - 后面是'.'或'e'则继续解析
        - 否则报错
    - '+'或'-'
        - 设置符号位
    - 1~9: 跳出前置特殊判断, 正常跑
- 在整形状态
    - 0~9跳过
    - '.'进入小数状态
    - 'e'进入科学记数法
    - 其他, 结束
- 在小数状态
    - 0~9跳过
    - 'e'进入科学记数法
    - '.'报错
    - 其他, 结束
        - 结束时判断结尾是不是'.'报错
        - 解析数字
- 在科学记数法状态
    - 0~9跳过
    - +/-跳过(设置标志位)
    - '.'报错
    - 'e'报错
    - 其他, 结束
        - 结束时判断结尾是不是'e'或者'.'报错
        - 解析数字

#### 计算:

接受几个状态: 正负性, 是否浮点, 是否科学记数法, 指数为正负性

### 字符串
- 遇到`"`, 开始识别字符串, 直到行尾, 忽略转义的\", 如果未匹配""则报错
- 字符串解码, 转义的\uXXXX为UTF-16, 以及一些特殊的转义符, 对非法转义符报错

### 对象
- 遇到`{`开始解析对象, `"str" : value`格式, 可有多组, 后跟随`,`, 最终都以`}`结尾, 期间忽略空白

    - 期间出现未预期的字符, 格式错误, 报错
    - 扫到文件尾仍未闭合的`{}`, 报错
    - 键或值出现的解析错误, 复用并返回 

### 数组
- 遇到`[`开始解析数组, 多个value, 中间`,`分隔, 最终以`]`结尾, 期间忽略空白
    - 未预期字符, 格式错误, 报错
    - 扫到文件尾仍未闭合的`[]`, 报错
    - 值出现解析错误, 复用并返回


## New json state machine for SAX

- Istream模型
    - 必须提供: `peek()`, `seek()`, `get()`
    - `peek()`
        - 仅查看读指针位置的字符, 不移动指针
    - `seek()`
        - 仅将指针向后移动一位
    - `get()`
        - 行为严格为一次`peek()`得到字符, 然后一次`seek()`移动指针, 再返回peek到的字符
    - 当指针位于max_idx+1, 也就是等于总长时, 再次`peek()`或`get()`必须返回'\0'以告知结束

- 调度函数
    - `0~9` `+/-`: 调用parse_number
        - 注意, parse_number由于没有确定的结束符, 将会多吃掉一个不属于它自己的字符才能判断是否结束
    - `"`: 字符串, 调用parse_string
    - `[`: 数组, 调用parse_array
    - `{`: 对象, 调用parse_object

- parse_number
    - start
        - `0`开头
            - 后面跟`.`或`e`
            - 后面跟其他, 返回0
        - `+/-`开头
            - 后面必须是数字
            - 否则报错
    - loop (get()读指针移动)
        - `.`
        - `e`
        - `0~9`
        - `+/-`
        - 其他, 结束, 解析数字
    - 检查数字字面量的结尾是否合法
