# 思路梳理

## json中的几种结构

### 特殊字面量
- `true`
- `false`
- `null`

### 数字
必然以`0~9` 或 `-` ASCII 开头, 中间可有且仅允许一个`e`和`.`, 点号必须在e前且非封闭

此处可以单独开一个函数(代号read_int)识别多位整数, 然后判断有没有`.`, 识别浮点数, 再判断有没有`e`, 识别科学记数法

json不允许数字前导零, 所以read_int在遇到第一个0时就返回, `0`后面必须只有`.`或`e`才能继续接数字, 否则报错 (0)

负数特殊处理, 打个标志位然后解析整数再转一下相反数就行

### 字符串
- 遇到`"`, 开始识别字符串, 直到行尾, 忽略转义的\", 如果未匹配""则报错 (1)
- 字符串解码, 转义的\uXXXX为UTF-16, 以及一些特殊的转义符, 对非法转义符报错 (2)
> 使用单独的字符串解析函数

### 对象
- 遇到`{`开始解析对象, `"str" : value`格式, 可有多组, 后跟随`,`, 最终都以`}`结尾, 期间忽略空白

    - 期间出现未预期的字符, 格式错误, 报错 (3)
    - 扫到文件尾仍未闭合的`{}`, 报错 (4)
    - 键或值出现的解析错误, 复用并返回 (...)

### 数组
- 遇到`[`开始解析数组, 多个value, 中间`,`分隔, 最终以`]`结尾, 期间忽略空白
    - 未预期字符, 格式错误, 报错 (5)
    - 扫到文件尾仍未闭合的`[]`, 报错 (6)
    - 值出现解析错误, 复用并返回 (...)
